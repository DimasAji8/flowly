## 4. Phase 1 — Auth (selesai)

### Backend
- Module `users` (UsersService).
- Module `auth`:
  - Strategi JWT access (Bearer header) & JWT refresh (body field).
  - DTOs: `RegisterDto`, `LoginDto`, `RefreshDto` dengan class-validator.
  - bcrypt rounds 12, JWT secret + refresh secret terpisah, expiry konfigurabel via env.
  - **Auto-create personal workspace + membership owner** dalam 1 transaksi atomik saat register.
  - **Auto-seed default wallet (Cash) + 8 kategori** (5 expense + 3 income) saat register.
  - Endpoint: `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh`, `GET /auth/me`.
  - Pesan error login generik (tidak bocorin "user ada/tidak").
- Common building blocks:
  - `@CurrentUser()` dan `@CurrentJwtPayload()` decorators
  - `JwtAuthGuard`, `JwtRefreshGuard`
  - Type `AuthUser`, `FlowlyJwtPayload` (rename untuk hindari konflik dengan `jsonwebtoken`)
- **WorkspaceGuard** (validasi member via header `X-Workspace-Id`) + `@CurrentWorkspace()` + `@WorkspaceRoles('owner'|'member')` decorator.
- Module `workspaces`:
  - `GET /workspaces` (list workspace user)
  - `GET /workspaces/current` (detail workspace dari header)

### Frontend
- `lib/auth-storage.ts` — wrapper localStorage untuk session.
- `lib/api-client.ts` — fetch wrapper dengan auto-inject `Authorization` & `X-Workspace-Id`, auto-refresh token saat 401, mutex agar tidak double-refresh.
- `services/auth.service.ts` — register/login/me.
- `store/auth.store.ts` (Zustand) — hydrate dari storage, setSession, clear, refreshMe.
- `hooks/use-auth.ts` — `useRequireAuth` (protected page) & `useRedirectIfAuthed` (auth page).
- `components/layout/auth-hydrator.tsx` — hidrasi store sekali di root.
- Halaman `/auth/login`, `/auth/register` (React Hook Form + Zod).
- Halaman `/dashboard` v1 (welcome + logout).
- Auto-redirect: belum login → login page, sudah login → dashboard.

### Verifikasi
- 8 skenario e2e: register baru, /auth/me, login, refresh, login wrong password, register duplicate, validasi DTO, /me tanpa token. Semua PASS.
- Unit test `app.controller.spec.ts` — 3 test PASS.

## 8. Phase 4 — Recurring (selesai)

### Backend
- Install `@nestjs/schedule`.
- Module `recurring`: full CRUD (5 endpoint).
- `RecurringService.runDueJobs(now)`:
  - Cari semua recurring `isActive=true && nextRunAt <= now`.
  - Untuk tiap due: dalam 1 transaksi DB atomik → buat `Transaction` (date = nextRunAt), update `Wallet.balance`, advance `nextRunAt` sampai melewati `now` (catch-up safe — server mati 3 hari recurring daily tidak menggandakan).
  - Owner workspace dijadikan `userId` transaksi.
  - Logging hasil eksekusi.
- `jobs/jobs.module.ts` + `recurring.job.ts` — `@Cron(EVERY_HOUR)`.

## 10. Hal-hal Lain Sepanjang Sesi

- **inotify limit issue** — diidentifikasi (default Ubuntu 65k watches), saran naikkan ke 524k.
- **Process zombie** dari testing background — diidentifikasi sebagai penyebab `ERR_CONNECTION_REFUSED` & `EADDRINUSE`, dibersihkan beberapa kali.
- **Hydration error** dengan pesan `<div hidden>` vs whitespace — diidentifikasi kemungkinan dari iframe simulator/preview tool, bukan dari kode aplikasi. Penanganan defensive: `suppressHydrationWarning` di root layout. Investigasi lengkap ditunda.
- **Turbopack** — tadinya tidak eksplisit, ditambahkan `--turbopack` di script `dev` & `build`.
- **`next/font/google` (Geist)** dihapus untuk sementara, ganti font system stack — menghindari hydration mismatch dengan font hash class.
- **`tsconfig.json` API** — `baseUrl` deprecated di TS 7.0, dihapus karena tidak terpakai.
- **Responsive** — sidebar desktop ≥768px, bottom nav mobile <768px, content max-width 480px (mobile) / 760px (desktop).
- **Process saya kill saat smoke test** — bukan crash. Kebijakan baru: hanya kill PID yang saya start sendiri, atau gabung ke instance user kalau port sudah aktif.

## 13. Cara Pakai

```bash
# 1. Install (sekali)
npm install

# 2. Setup env (sekali)
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env.local
# edit apps/api/.env, ganti PASSWORD di DATABASE_URL dengan password Postgres lokal

# 3. Migration sudah dijalankan, tidak perlu re-run
# Kalau mau reset: npm run db:migrate (di apps/api)

# 4. Jalankan dev (web + api bersamaan)
npm run dev
```
